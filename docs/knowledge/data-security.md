# Data Security Approach

Cadre evaluates data security as part of the design of each AI solution. That includes considering what information is sent to a model, who can access the system, provider data-handling terms, retention, logging, encryption, and deployment requirements.

The correct approach depends on your systems, data sensitivity, and regulatory obligations. Security is treated as a design consideration for each solution rather than a single fixed policy applied everywhere.

Questions about certifications, compliance status, contractual commitments, data residency, or a specific organization's architecture are handled directly by the Cadre team rather than in this chat. For a security review or organization-specific requirements, Cadre can connect you with the appropriate team.
